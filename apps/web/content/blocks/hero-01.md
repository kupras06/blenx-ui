## Overview

Hero-01 is a marketing section block that serves as the primary above-the-fold content for landing pages and product sites. It presents a centered or left-aligned headline, supporting subtext, two call-to-action buttons with visual hierarchy, and an image placeholder that can hold a product screenshot, illustration, or video embed. The block is designed as a full-width section rather than a contained card, spanning the entire viewport width with optional background color or gradient support. The headline uses your site's largest heading token to create immediate visual impact, while the subtext provides the value proposition in one to two sentences. The two CTA buttons establish a clear primary action — such as "Get Started" — and a secondary action — such as "Learn More" — guiding users toward conversion without overwhelming them with choices.

## When To Use

Use hero-01 as the opening section of any marketing page, product landing page, or campaign site where you need to capture attention and drive a specific user action. It works well for SaaS products, developer tools, mobile apps, and informational sites that follow a standard landing-page hierarchy. The block is not suitable for interior pages like dashboards, admin panels, or documentation, where a prominent hero section would compete with functional content. Avoid using hero-01 if your brand requires a more experimental or asymmetrical hero layout, as the block follows a conventional centered or left-aligned structure with the image on the right. It also should not be used when you need multiple headline variants or dynamic hero content that changes based on user segments.

## Customization

The `headline` prop accepts the primary heading text, which can include inline spans for gradient text or emphasis through your theme's tokens. The `subtext` prop provides the supporting paragraph below the headline. The `primaryCta` and `secondaryCta` props each accept `label` and `href` (or `onClick`) properties, configuring the two action buttons. The `imageSrc` and `imageAlt` props set the hero image source and alt text; omitting `imageSrc` renders an empty placeholder with a dashed border for development previews. The `imagePosition` prop accepts `"right"` or `"bottom"` to control whether the image sits beside or below the text content, with `"right"` being ideal for desktop and `"bottom"` for mobile-first layouts. The `background` prop accepts a theme color token or gradient string to set the section background.

## Accessibility Notes

The hero section uses a semantic `<section>` element with an `aria-label` describing its purpose as a hero or banner. The heading is an `<h1>` element, establishing it as the primary page heading for document outline purposes. The image includes descriptive `alt` text — if the image is purely decorative, `alt` should be set to an empty string. The two CTA buttons have descriptive text that communicates the action — "Get started for free" carries more meaning than "Learn more" for screen reader users. Focus order places the primary CTA before the secondary CTA, matching the visual hierarchy. The section has a `tabindex="-1"` on the container to allow programmatic focus management if the hero is part of a route transition.

## Composition

Hero-01 composes a full-width `Section` component with a constrained inner `Container`. Inside, a `Grid` or `Flex` layout separates the text stack from the image. The text stack contains a `Heading` (at the `h1` level), a `Text` component for the subtext, and a `Button` group rendered as a `Flex` row. The image area uses an `Image` component with responsive sizing, falling back to a `Placeholder` component (a dashed `Box`) when no source is provided. The `Button` components use the `primary` and `secondary` variants to establish visual weight. An optional `Tagline` component can be added above the heading for branding text like "Introducing Blenx 2.0."

## Best Practices

Keep the headline under 10 words and the subtext under 25 words — a hero section's job is to orient and motivate, not to explain in depth. Make the primary CTA a single, specific action that represents the core conversion goal of the page, such as "Start your free trial" rather than "Submit." Ensure the hero image communicates the product's value visually — a screenshot of the product interface typically outperforms abstract illustrations. Maintain strong contrast between the text and the background, especially when using gradient or image backgrounds; overlay gradients or darken images as needed to ensure the headline remains readable. Test the layout at all breakpoints since the hero is the first thing users see on every device.

## Limitations

Hero-01 supports exactly two CTAs — if your marketing strategy requires a single CTA or three or more, this block will not fit without modification. The image placeholder pattern (dashed border for missing source) is development-only and should never ship to production without a real image. The block does not include any animation or parallax effects on scroll — it renders statically. There is no built-in A/B testing or personalization capability; serving different hero variants to different user segments requires external logic. The section is full-width but the content is constrained — if you need a true full-bleed image hero with text overlaid, you will need to adjust the container behavior.
